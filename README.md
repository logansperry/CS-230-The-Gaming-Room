## 🧩 Client Summary and Requirements
The client, The Gaming Room, requested the expansion of their Android-based game, *Draw It or Lose It*, into a scalable, web-based application. The system needed to support cross-platform access for users on desktop and mobile devices while maintaining high performance.

Key requirements included:
- Support for thousands of concurrent users
- Cross-platform compatibility
- Secure authentication
- Persistent data storage
- Low latency and high availability

To meet these needs, I designed a **client-server architecture** using a web-based interface supported by a **Linux-based server environment** and RESTful APIs.
